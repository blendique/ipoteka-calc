# Поля по СК для мульти-коннектора (Альфа, Абсолют, СОГАЗ)

Правило: если поле обязательно хотя бы в одной СК — показываем в форме. При отправке в конкретную СК передаём только то, что она принимает; остальное не отправляем.

## 8. Рекомендации по UI (мульти-коннектор)

1. **Выбор СК:** в начале — чекбоксы «Альфа», «Абсолют», «СОГАЗ». Показывать только поля, нужные хотя бы одной выбранной СК.
2. **Дропдауны с разными значениями:** один общий список вариантов, у каждого варианта справа лейбл(ы) СК (Альфа / Абсолют / СОГАЗ) и иконка «?» с тултипом — описание пункта. При выборе СК маппим значение в код этой СК.
3. **Радио (пол, тип объекта, тип страхования):** рядом с подписью — маленький лейбл «Альфа», «Абсолют», «СОГАЗ» (кому нужно); справа от опции — «?» с подсказкой.
4. **Обязательность:** помечать «*» только если поле обязательно хотя бы в одной из выбранных СК. В тултипе можно писать: «Обязательно для: Альфа, СОГАЗ».
5. **Отправка:** при вызове API конкретной СК формировать тело только из полей этой СК; пустые или неиспользуемые не передавать.
