# Рекомендации по типам полей (коннектор ипотечного страхования)

## Принципы
- **Дропдаун** — когда значения из справочника API или фиксированный короткий список.
- **Текстовый ввод** — произвольный текст, номера документов, адреса.
- **Число** — суммы, проценты, сроки с валидацией и подсказками лимитов.
- **Дата** — единый date picker (YYYY-MM-DD).
- **Чекбокс/переключатель** — да/нет, признаки (горючесть, согласие супруга и т.д.).

---

## 1. Общие и ипотечный договор

| Поле | Тип элемента | Обоснование |
|------|----------------|-------------|
| **beginDate** | **Date picker** (календарь) | Одна дата, ограничение: от сегодня до +3 мес. Подсказка под полем. |
| **bankName** | **Дропдаун (поиск)** | Справочник банков (п.2). Желательно поиск по названию. Показывать bankName, сохранять id/code. |
| **creditValue** | **Число** (input type="number" или маска) | Лимиты 100 000 – 20 000 000. Разделитель тысяч, суффикс "₽". |
| **rate** | **Число** (проценты) | 0–100, один знак после запятой. Для Райффайзенбанка — подсказка "только 10%". |
| **termInMonth** | **Дропдаун или число** | Месяцы. Лимиты из п.7 (minInsuranceTerm – maxInsuranceTerm). Дропдаун с типовыми сроками (12, 24, 36… до макс.) удобнее. |
| **dateCreditDoc** | **Date picker** | Дата подписания кредитного договора. |
| **numberCreditDoc** | **Текстовое поле** | Номер договора у банка. Только при сохранении. |
| **address (кредит)** — state | **Дропдаун или автокомплит** | Регионы РФ. Справочник или API (ФИАС/КЛАДР). |
| **address (кредит)** — city | **Автокомплит** | Города, желательно зависящие от выбранного региона. |

## 2. Объект страхования

| Поле | Тип элемента | Обоснование |
|------|----------------|-------------|
| **buildingType** | **Дропдаун** | FLAT / ROOM / APARTMENTS. Варианты ограничивать по выбранному банку (п. спецификации). |
| **saleType** | **Радио-кнопки или дропдаун** | Первичный рынок / Вторичный рынок (2 варианта). |
| **address** — state, city | Как для кредита | **Дропдаун региона + автокомплит города**. |
| **address** — street, house, building, apartment | **Текстовые поля** | Улица, дом, корпус, квартира. Опционально. |
| **propertyArea** | **Число** | Площадь, м². Только при сохранении. Одна десятичная, мин 1. |
| **propertyRisk.goruch** | **Чекбокс/переключатель** | "Стены и перекрытия горючие" (да/нет). |
| **propertyRisk.year** | **Число или дропдаун** | Год постройки. Ограничение: не старше 1960. Дропдаун год от 1960 до текущего. |
| **propertyRisk.capitalRepairExists** | **Чекбокс** | "Был капремонт/перепланировка". |
| **propertyRisk.insuranceBaseAmount** | **Число** | Рыночная стоимость, ₽. Аналогично creditValue по формату. |
| **propertyRisk.kadastr** | **Текстовое поле** | Кадастровый номер. Подсказка формата (например 77:05:0005005:123). |
| **propertyRisk.repairWorkInProgress** | **Чекбокс** | "Сейчас ведутся ремонтные работы". |
| **propertyRisk.poolExists** | **Чекбокс** | "Есть бассейн". |
| **titleRisk.*** (все булевы) | **Чекбоксы** | Сделка по доверенности, собственники 18-/65+, юрлица, в собственности <3 лет, нет согласия супруга. |
| **titleRisk.termInMonth** | **Дропдаун/число** | Срок по титулу, не больше общего срока. |
| **titleRisk.ownershipConfirmationDocument** | **Дропдаун** | Справочник п.6 (документ, подтверждающий право). id + name. |
| **titleRisk.ownershipRestriction** | **Дропдаун** | Справочник п.5 (ограничение права). id + name. |
| **titleRisk.insuranceBaseAmount** | **Число** | Рыночная стоимость для титула. |

## 3. Страхователь и созаёмщик

| Поле | Тип элемента | Обоснование |
|------|----------------|-------------|
| **lastName, firstName, middleName** | **Текстовые поля** | ФИО. middleName опционально. |
| **gender** | **Радио-кнопки** | М / Ж (M / F). Два варианта. |
| **dateOfBirth** | **Date picker** | С проверкой возраста по лимитам банка (п.7). |
| **share** | **Число** | Доля, %. 0–100, при одном заёмщике подсказка "обычно 100%". |
| **email** | **Текстовое поле** type="email" | Email. |
| **phone** | **Текстовое поле** (маска телефона) | +7 (___) ___-__-__. |
| **address** (регистрация) | Как выше | state, city — дропдаун/автокомплит; street, house, building, apartment — текст. |
| **personDocument.seria** | **Текстовое поле** (маска 4 цифры) | Серия паспорта. |
| **personDocument.number** | **Текстовое поле** (маска 6 цифр) | Номер паспорта. |
| **personDocument.dateOfIssue** | **Date picker** | Дата выдачи. |
| **personDocument.issuedBy** | **Текстовое поле** | Кем выдан (развёрнутый текст). |
| **personDocument.issuedByUnitCode** | **Текстовое поле** (маска 000-000) | Код подразделения. |
| **resident** | **Чекбокс** | "Резидент РФ" (по умолчанию включено). |
| **lifeRisk.illness** | **Чекбокс** | "Есть хронические заболевания". |
| **lifeRisk.professionId** | **Дропдаун (поиск)** | Справочник профессий п.4. Показывать name, сохранять id (+ profession текст при необходимости). |
| **lifeRisk.sportId** | **Дропдаун (поиск)** | Справочник спорта п.3. name + id. |

## 4. Агентский блок (часто скрыт или предзаполнен)

| Поле | Тип элемента | Обоснование |
|------|----------------|-------------|
| **managerId** | **Число или дропдаун** | Если у партнёра список менеджеров — дропдаун, иначе число. |
| **agentContractId** | **Число** | Обычно одно значение на партнёра — скрытое или одно поле. |
| **departmentId**, **channelSaleId** | **Скрытые или число** | По умолчанию из конфига (2141, 6925629). |
| **promoCode** | **Текстовое поле** | Опционально, одно поле внизу формы или в "Доп. опциях". |

## 5. Сводка по типам

| Тип | Где использовать |
|-----|-------------------|
| **Дропдаун** | Банк, тип/вид имущества, регион, профессия, спорт, документы титула, срок (мес). |
| **Дропдаун с поиском** | Банк, профессия, спорт при длинном списке. |
| **Текстовое поле** | ФИО, email, номера документов, улица, дом, корпус, квартира, кадастровый номер. |
| **Число** | Суммы (кредит, рыночная стоимость), ставка %, доля %, площадь, год постройки. |
| **Date picker** | beginDate, dateCreditDoc, dateOfBirth, dateOfIssue. |
| **Чекбокс/переключатель** | Все булевы поля (горючесть, капремонт, болезни, согласие супруга и т.д.). |
| **Радио** | Пол (М/Ж), первичка/вторичка (если не дропдаун). |

---

## 6. UX-рекомендации

- Показывать блоки **по шагам (wizard)** или **аккордеоном**: 1) Банк и кредит, 2) Объект, 3) Страхователь, 4) Созаёмщик (если есть), 5) Доп. опции.
- После выбора банка — запросить п.7 (ограничения) и п.8 (поля анкеты) и показывать только нужные поля, помечая обязательные.
- Под полями с лимитами выводить подсказки: "От 100 000 до 20 000 000 ₽", "Возраст 18–50 лет при страховании жизни".
- Валидация на клиенте по тем же лимитам, что в п.7 и п.8.
